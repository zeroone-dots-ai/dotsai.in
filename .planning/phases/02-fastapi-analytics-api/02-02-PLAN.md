    **analytics/app/routes/events.py** - The core ingest endpoint:

    - Import: `APIRouter`, `BackgroundTasks`, `Depends`, `Request`, `Response` from FastAPI
    - Import: `AsyncSession` from sqlalchemy, `select` from sqlalchemy
    - Import: `async_session` from app.database (the sessionmaker, NOT a dependency -- background tasks create their own session)
    - Import: `verify_token` from app.dependencies
    - Import: `EventIn` from app.schemas
    - Import: `Visitor`, `Event` from app.models
    - Import: `hashlib`, `datetime`
    - Import: `settings` from app.config
    - Import: `limiter` from app.main (or pass via app.state)

    **Fingerprinting function:**
    ```python
    def make_fingerprint(ip: str, ua: str, salt: str) -> str:
        daily = f"{salt}:{datetime.date.today().isoformat()}:{ip}:{ua}"
        return hashlib.sha256(daily.encode()).hexdigest()
    ```
    - Daily-rotating: same visitor gets different hash each day
    - Raw IP NEVER stored -- only the hash

    **Background write function** (`async def _write_event`):
    - CRITICAL: Must create its OWN async session using `async with async_session() as db:`
    - Do NOT accept a db session parameter -- the request session is closed after 202 response
    - Extract real IP from X-Real-IP header (set by nginx), fallback to request.client.host
    - Compute fingerprint from IP + User-Agent + salt
    - Upsert visitor: SELECT by fingerprint. If exists, update last_seen. If not, INSERT new visitor with fingerprint, country=None, city=None, referrer from payload.
    - INSERT event: visitor_id, site, page, event_name, properties from payload
    - Commit. Handle exceptions with try/except -- log errors but never crash (fire-and-forget).

    **POST /events endpoint:**
    - Decorator: `@router.post("/events", status_code=202)`
    - Apply rate limit: `@limiter.limit("100/minute")` -- uses the get_real_ip key function from main.py
    - Parameters: `request: Request`, `event: EventIn`, `background_tasks: BackgroundTasks`, `token: str = Depends(verify_token)`
    - Body: Extract IP and UA from request, add background task `_write_event(event, ip, ua)`, return `Response(status_code=202)`
    - The 202 returns IMMEDIATELY -- DB write happens asynchronously

    **Rate limiter access:** Import `limiter` from app.main or access via `request.app.state.limiter`. Use the decorator approach: `@limiter.limit("100/minute")`.

    Note on slowapi: The limiter needs the `request` parameter in the endpoint signature to extract the key. Ensure the endpoint has `request: Request` as the first parameter.
