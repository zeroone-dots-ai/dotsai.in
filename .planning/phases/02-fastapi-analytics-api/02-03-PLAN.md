    **Step 3 -- Create env file (chmod 600, outside repo):**

    ```bash

    TOKEN=$(openssl rand -hex 32)
    SALT=$(openssl rand -hex 16)

    cat > /opt/services/.env.analytics << EOF
    DATABASE_URL=postgresql+asyncpg://dotsai:<password>@postgres:5432/dotsai
    ANALYTICS_WRITE_TOKEN=$TOKEN
    FINGERPRINT_SALT=$SALT
    EOF
    chmod 600 /opt/services/.env.analytics
    ```

    The `<password>` must be replaced with the actual dotsai DB password from /opt/services/.secrets/db_password.txt (created in Phase 1).

    Print the ANALYTICS_WRITE_TOKEN value -- it will be needed in Plan 02-04 for the browser snippet. Store it securely (the user will need it).

    **Step 4 -- Add analytics service to docker-compose.yml:**

    Add this service to /opt/services/docker-compose.yml:

    ```yaml
    analytics:
      build: ./dotsai.in/analytics
      container_name: analytics
      restart: always
      env_file:
        - .env.analytics
      depends_on:
        postgres:
          condition: service_healthy
      networks:
        - dotsai_internal
      deploy:
        resources:
          limits:
            memory: 128M
    ```

    No `ports:` -- nginx proxies via internal network. The `build:` path points to analytics/ inside the cloned repo.

    **Step 5 -- Build and start analytics:**

    ```bash
    cd /opt/services
    docker compose build analytics
    docker compose up -d analytics
    # Wait for startup + Alembic migration
    sleep 5
    docker logs analytics --tail 20
    # Verify: should show "alembic upgrade head" output then "Uvicorn running on 0.0.0.0:8000"
    ```

    Verify tables created:
    ```bash

    # Should show: visitors, events, alembic_version
    ```

    **Step 6 -- nginx server block for api.dotsai.in:**

    Create /opt/services/nginx/conf.d/api.dotsai.in.conf (inside the nginx Docker volume):

    First, create HTTP-only block for certbot challenge:
    ```nginx
    server {
        listen 80;
        server_name api.dotsai.in;

        location /.well-known/acme-challenge/ {
            root /var/www/certbot;
        }

        location / {
            return 301 https://$host$request_uri;
        }
    }
    ```

    Reload nginx: `docker exec nginx nginx -s reload`

    **Step 7 -- SSL certificate (staging first, then production):**

    ```bash
    # STAGING first -- safe to retry without rate limits
    certbot certonly \
      --webroot \
      -w /opt/services/nginx/certbot/ \
      -d api.dotsai.in \
      --staging \
      --agree-tos \
      --email aamdhanee.dev@gmail.com \
      --non-interactive

    # If staging succeeds, get production cert:
    certbot certonly \
      --webroot \
      -w /opt/services/nginx/certbot/ \
      -d api.dotsai.in \
      --agree-tos \
      --email aamdhanee.dev@gmail.com \
      --non-interactive \
      --force-renewal
    ```

    ADAPT the certbot approach based on Step 1 findings. If certbot runs inside Docker or uses a different webroot, adjust accordingly. Check how existing dotsai.in certs were obtained and follow the same pattern.

    **Step 8 -- Add HTTPS block to nginx:**

    Update /opt/services/nginx/conf.d/api.dotsai.in.conf to add the SSL server block:
    ```nginx
    server {
        listen 443 ssl http2;
        server_name api.dotsai.in;

        ssl_certificate /etc/letsencrypt/live/api.dotsai.in/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/api.dotsai.in/privkey.pem;

        resolver 127.0.0.11 valid=10s ipv6=off;
        set $upstream_analytics http://analytics:8000;

        location / {
            proxy_pass $upstream_analytics;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            # NO CORS headers -- FastAPI CORSMiddleware handles it (REQ-08.5)
        }
    }
    ```

    The `resolver 127.0.0.11` directive is REQUIRED for Docker DNS resolution (REQ-08). The `set $upstream_analytics` variable forces re-resolution on each request.

    Adjust SSL cert paths if they differ from the standard certbot location (check Step 1 findings).

    Reload nginx: `docker exec nginx nginx -s reload`
