  <action>
    Initialize Alembic with the async template inside analytics/:

    ```bash
    cd analytics && alembic init -t async alembic
    ```

    **Edit analytics/alembic.ini:**
    - Set `sqlalchemy.url = postgresql+asyncpg://dotsai:password@postgres:5432/dotsai` (placeholder -- overridden by env.py at runtime)
    - Set `version_path_separator = os` for cross-platform compatibility

    **Edit analytics/alembic/env.py** (critical -- must handle analytics schema correctly):
    - Import: `from app.models import Base` and `from app.config import Settings`
    - Set `target_metadata = Base.metadata`
    - In `run_migrations_online()`, override the URL from Settings().DATABASE_URL if available (fall back to alembic.ini for local dev)
    - In `do_run_migrations(connection)`:
      - Set `include_schemas=True` in `context.configure()`
      - Set `version_table_schema="analytics"` (stores alembic_version table in analytics schema, not public)
      - Before `context.run_migrations()`, execute: `SET search_path TO analytics, public`
    - This is the CRITICAL wiring -- if `include_schemas` or `version_table_schema` are wrong, tables land in public schema instead of analytics

    **Create migration manually** at analytics/alembic/versions/001_create_analytics_tables.py:
    - revision id: generate a random 12-char hex
    - `upgrade()`:
      - `op.execute("CREATE SCHEMA IF NOT EXISTS analytics")`
      - Create analytics.visitors table: id UUID pk default gen_random_uuid(), fingerprint VARCHAR(64) with index, country VARCHAR(2), city VARCHAR(128), referrer TEXT, first_seen TIMESTAMPTZ default now(), last_seen TIMESTAMPTZ default now()
      - Create analytics.events table: id BIGSERIAL pk, timestamp TIMESTAMPTZ default now(), visitor_id UUID FK to analytics.visitors.id, site VARCHAR(128) NOT NULL, page TEXT NOT NULL, event_name VARCHAR(64) NOT NULL, properties JSONB default '{}'
      - Create index on events(event_name)
      - Create index on events(site, timestamp)
    - `downgrade()`: drop analytics.events, drop analytics.visitors, drop schema analytics

    Write the migration using `op.create_table()` with `schema="analytics"` parameter, NOT raw SQL for table creation. The `CREATE SCHEMA IF NOT EXISTS` can be raw SQL since Alembic has no schema-create op.
  </action>
