**Step 1 — VPS Resource Audit (read-only, no changes)**


**Decision logic:**
- If `free -h` shows less than 3GB available (MemAvailable) AND no swap exists: configure 2GB swap.
- If Docker Compose v2 is NOT available: install it (`apt-get install docker-compose-plugin` or equivalent).
- Note the existing nginx container name, image, network, and ports for compose.yaml integration.

**Step 2 — Swap Setup (ONLY if needed per Step 1)**

If swap is needed:
```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
echo 'vm.swappiness=10' >> /etc/sysctl.conf
sysctl -p
```
Verify with `swapon --show` and `free -h`.

**Step 5 — Create schema init script**

Write `/opt/services/init-db/01-create-schemas.sql`:
```sql
\set ON_ERROR_STOP on

CREATE SCHEMA IF NOT EXISTS calcom;
CREATE SCHEMA IF NOT EXISTS analytics;

DO $$
BEGIN
  ASSERT (SELECT COUNT(*) FROM information_schema.schemata
          WHERE schema_name IN ('calcom', 'analytics')) = 2,
         'Expected both calcom and analytics schemas to exist';
END $$;
```

**Step 6 — Create or update compose.yaml**

CRITICAL: If an existing compose.yaml (or docker-compose.yml) exists, you MUST preserve the existing nginx service definition exactly. Do NOT remove or modify the nginx service. Add the postgres service alongside it.

If NO compose file exists but nginx runs via `docker run`, you must:
1. Inspect the running nginx container: `docker inspect nginx` (get image, volumes, ports, networks, env, restart policy)
2. Recreate that exact configuration as a service in compose.yaml
3. Add postgres alongside it

The compose.yaml must contain:
```yaml
services:
  # ... existing nginx service preserved exactly ...

  postgres:
    image: postgres:17-alpine
    container_name: postgres
    restart: always
    shm_size: 256mb
    environment:
      POSTGRES_DB: dotsai
      POSTGRES_USER: dotsai_admin
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
      POSTGRES_INITDB_ARGS: "--data-checksums"
    secrets:
      - db_password
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./init-db:/docker-entrypoint-initdb.d:ro
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U dotsai_admin -d dotsai"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 60s
    networks:
      - internal
    # No ports: section — internal Docker network only

volumes:
  pgdata:
    name: dotsai_pgdata

secrets:
  db_password:
    file: ./.secrets/db_password.txt

networks:
  internal:
    name: dotsai_internal
  # ... preserve any existing networks nginx uses ...
```

Ensure nginx is also on the `internal` network (add it to nginx's networks list) so future services can reach both.

**Step 7 — Bring up postgres**


Wait for healthy status:
```bash
# Poll until healthy (max 90 seconds)
for i in $(seq 1 18); do
  STATUS=$(docker inspect --format='{{.State.Health.Status}}' postgres 2>/dev/null)
  echo "Attempt $i: $STATUS"
  if [ "$STATUS" = "healthy" ]; then break; fi
  sleep 5
done
```

Verify nginx is still running:
