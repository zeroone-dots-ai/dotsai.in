**Step 4 — Create analytics/app/routes/webhooks.py:**

Create the webhook route module:

```python
import hashlib
import hmac
from fastapi import APIRouter, Request, HTTPException, BackgroundTasks
from sqlalchemy import select, update
from app.config import get_settings
from app.database import async_session_factory
from app.models import Booking

router = APIRouter(prefix="/webhooks", tags=["webhooks"])
```

POST /webhooks/calcom endpoint logic:
1. Read raw body bytes via `await request.body()`
2. Get `X-Cal-Signature-256` header (strip optional "sha256=" prefix if present)
3. If CAL_WEBHOOK_SECRET is not set, return 500 with {"error": "webhook secret not configured"}
4. Compute HMAC-SHA256 of raw body using CAL_WEBHOOK_SECRET as key
5. Compare with `hmac.compare_digest()` -- if mismatch, return 403 with {"error": "invalid signature"}
6. Parse JSON body
7. Extract `triggerEvent` field from payload (BOOKING_CREATED, BOOKING_CANCELLED, BOOKING_RESCHEDULED)
8. Extract booking data from `payload` field:
   - `cal_booking_id` from payload["bookingId"] or payload["id"]
   - `event_type` from payload["eventTitle"] or payload["type"]["slug"]
   - `attendee_name` from payload["responses"]["name"]["value"] or payload["attendeeName"]
   - `attendee_email` from payload["responses"]["email"]["value"] or payload["attendeeEmail"]
   - `start_time` from payload["startTime"]
   - `end_time` from payload["endTime"]
   - Use try/except with fallback for each field -- Cal.com payload structure varies by event type
9. Background task (using own async session per established pattern from 02-02):
   - BOOKING_CREATED: Insert new Booking row with status="created"
   - BOOKING_CANCELLED: `UPDATE analytics.bookings SET status='cancelled' WHERE cal_booking_id = :id`. If no row found, insert with status="cancelled" (idempotent).
   - BOOKING_RESCHEDULED: `UPDATE analytics.bookings SET status='rescheduled', start_time=:new_start, end_time=:new_end WHERE cal_booking_id = :id`. If no row found, insert with status="rescheduled".
   - Always store raw_payload (the full JSON body)
10. Return 200 with {"status": "ok"}

IMPORTANT: The webhook endpoint does NOT use Bearer auth (no ANALYTICS_WRITE_TOKEN check). It uses HMAC-SHA256 via X-Cal-Signature-256 header instead. Make sure the bot filter middleware in main.py does NOT apply to /webhooks/ paths -- check the existing BotFilterMiddleware scope (it currently only filters POST /events).
