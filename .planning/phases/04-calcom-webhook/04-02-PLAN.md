<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Configure Cal.com webhook in admin dashboard</name>
  <action>
The user must configure the webhook in Cal.com's admin UI. Claude cannot access Cal.com's authenticated dashboard.

**Steps for the user:**

1. Go to https://app.cal.com/settings/developer/webhooks
2. Click "New Webhook" (or "Add")
3. Configure:
   - **Subscriber URL:** `https://api.dotsai.in/webhooks/calcom`
   - **Secret:** Paste the CAL_WEBHOOK_SECRET value from Task 1
   - **Event triggers:** Select these three:
     - BOOKING_CREATED
     - BOOKING_CANCELLED
     - BOOKING_RESCHEDULED
   - **Payload template:** Leave as default (JSON)
   - **Active:** Enable
4. Click "Save" / "Create Webhook"

**Verify webhook was created:**
- The webhook should appear in the list with status "Active"
- The URL should show `https://api.dotsai.in/webhooks/calcom`
  </action>
  <resume-signal>Type "webhook configured" when the Cal.com webhook is saved and active.</resume-signal>
</task>
