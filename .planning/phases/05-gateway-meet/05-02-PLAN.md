Deploy meet.dotsai.in to the VPS. This follows the EXACT same pattern as api.dotsai.in (02-03-SUMMARY.md).

**Step 1: Copy meet page to VPS**
```bash
scp -r public/meet/ root@72.62.229.16:/opt/services/nginx/html/dotsai.in/meet/
```

**Step 2: Create nginx server block for meet.dotsai.in**
SSH to VPS and create `/opt/services/nginx/conf.d/meet.dotsai.in.conf`:

```nginx
server {
    listen 80;
    server_name meet.dotsai.in;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}
```

**Step 3: Reload nginx to serve HTTP (needed for certbot)**
```bash
ssh root@72.62.229.16 "docker exec nginx nginx -s reload"
```

**Step 4: Issue SSL certificate with certbot**
Same pattern as api.dotsai.in:
```bash
ssh root@72.62.229.16 "certbot certonly --webroot -w /opt/services/certbot-webroot -d meet.dotsai.in --config-dir /opt/services/nginx/certs --non-interactive --agree-tos -m aamdhanee.dev@gmail.com"
```

**Step 5: Update nginx conf to HTTPS**
Replace `/opt/services/nginx/conf.d/meet.dotsai.in.conf` with:

```nginx
server {
    listen 80;
    server_name meet.dotsai.in;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name meet.dotsai.in;

    ssl_certificate     /etc/letsencrypt/live/meet.dotsai.in/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/meet.dotsai.in/privkey.pem;

    root /usr/share/nginx/html/dotsai.in/meet;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
}
```

Note: The nginx container maps `/opt/services/nginx/html` to `/usr/share/nginx/html` and `/opt/services/nginx/certs` to `/etc/letsencrypt`. Verify these mappings match docker-compose.yml on VPS before proceeding.

**Step 6: Reload nginx**
```bash
ssh root@72.62.229.16 "docker exec nginx nginx -s reload"
```

curl -I https://meet.dotsai.in
# Expect: HTTP/2 200, valid SSL
